# План разработки визуализации

## Текущее состояние

Инструмент генерирует:
- `report.json` — машиночитаемый
- `report.md` — текстовая таблица before/after
- `report.html` — минималистичная HTML-таблица (без графиков)

**Проблема:** таблица чисел не даёт мгновенного понимания. Разница между `churn_rate: 0.18` до и `0.31` после — это много или мало? Хорошо или плохо?

---

## Цель визуализации

Дать команде возможность **за 30 секунд** понять:
1. Какие метрики изменились существенно.
2. В какую сторону (улучшение / деградация).
3. Насколько надёжен вывод.

---

## Фазы разработки

### Фаза V1 — Статичный HTML-дашборд (минимальный)

**Объём:** 1–2 дня работы
**Подход:** расширить существующий `report.html.j2`
**Зависимости:** только Jinja2 + Chart.js (CDN, без установки)

#### Что добавить в `report.html`:

**1. Bar chart "Before vs After" для каждой метрики**

```
mean_lines:  [===== 45 |========== 78]   +73%
churn_rate:  [== 0.18  |==== 0.31]       +72%  ⚠
test_ratio:  [=== 0.22 |======= 0.41]    +86%  ✓
```

Цветовая схема:
- Зелёный = улучшение (больше тестов, меньше churn)
- Красный = деградация (больше churn, больше hot-файлов)
- Серый = нейтральное изменение

**2. Confidence gauge**

Полукруглый индикатор: low / medium / high — сразу ясно, насколько доверять цифрам.

**3. Timeline заголовок**

```
[Jan 2024]───────●───────[Jul 2024]
               Jun 1
            Adoption date
```

#### Изменения в коде:

- `ai_adoption_diff/report/templates/report.html.j2` — добавить Chart.js из CDN и новые секции
- `ai_adoption_diff/report/renderer.py` — добавить вычисление `delta_pct` и `direction` для каждой метрики
- Новые поля в `MetricRow`: `delta_pct: float | None`, `direction: str` (up/down/neutral)

### Фаза V3 — CLI-команда `ai-diff serve` (опционально)

**Объём:** 2–3 дня
**Подход:** минимальный HTTP-сервер на `http.server` + автооткрытие браузера

```bash
ai-diff analyze --repo . --date 2024-06-01 --format html --output ./reports
ai-diff serve --output ./reports
# Открывает http://localhost:8080/report.html
```

Полезно для локального просмотра без копирования файлов.

### Фаза V4 — Multi-repo сравнение (долгосрок)

**Объём:** 1+ неделя
**Идея:** сравнивать несколько репозиториев на одном дашборде.

```bash
ai-diff compare \
  --repos repo1:cursor repo2:copilot repo3:none \
  --date 2024-06-01
```

Выводит grouped bar chart — одна группа баров на каждую метрику, по одному бару на репозиторий.

Полезно для команд с несколькими проектами или для сравнения разных AI-инструментов.

## Технические решения

### Почему Chart.js, а не D3.js?

- Chart.js даёт нужные типы графиков (bar, gauge, scatter) с минимальным кодом
- Работает без сборки (CDN или inline)
- Достаточно для CLI-инструмента

D3.js избыточен для данной задачи — пригодится только если понадобится нестандартная визуализация (например, flame graph по директориям).

### Почему не React/Vue?

Инструмент генерирует статичные файлы. Добавление JS-фреймворка означает шаг сборки (webpack/vite), что нарушает принцип "zero runtime dependencies" для конечного пользователя. Single-file HTML с inline Chart.js работает везде без установки.

### Хранение данных в HTML

Для V2 данные коммитов нужно передать в шаблон. Подход:

```html
<script>
const REPORT_DATA = {{ report_json | tojson }};
</script>
```

Jinja2 сериализует `AnalysisReport.model_dump()` в JSON прямо в шаблон. Файл остаётся self-contained.
